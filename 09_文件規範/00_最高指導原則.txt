🎯 專案目標
-----------
提供國家考試規格的線上模擬測驗平台，支援學科、術科、計算練習三種模式。
使用者無需安裝任何軟體，開啟瀏覽器即可使用。

🏗️ 技術架構
-----------
- 前端：純 HTML5 + JavaScript（無框架）
- 後端：FastAPI (Python 3.11)
- 資料庫：Supabase (PostgreSQL)
- 部署：Render (雲端)
- 版本：10.2.0

📁 目錄結構
-----------
01_前端資源/     - 靜態資源（CSS、圖片、設定）
02_前端服務/     - 前端服務模組（API、公式引擎、計算機、白板、鍵盤）
03_核心邏輯/     - 核心業務邏輯（題型規範、驗證器、解析引擎、渲染器）
04_應用層/       - 應用入口（index.html、系統主頁面.html）
05_後端伺服器/   - 後端 API（FastAPI）
06_資料庫/       - 資料庫檔案（題庫 JSON、法規資料）
09_文件規範/     - 文件規範

🔧 核心原則
-----------
1. 格式統一原則（v10.2.0 新增）
   - 題庫、法規、智慧化解析引擎三者格式必須完全對應
   - 所有 8 大題型使用統一欄位結構，不需要的欄位保留空白不刪除
   - 題庫的 law 欄位必須能正確對應到法規資料庫

2. 雙環境支援：本地開發 + 雲端部署（Render）
   - 本地開發：http://localhost:8001
   - 雲端部署：https://*.onrender.com

3. API 統一設定：所有 API 請求透過 00_API統一設定.js
   - 自動偵測環境（localhost / onrender.com）
   - 自動切換 API 網址

4. 統一答案儲存：使用 window.G_userAnswers
   - 所有渲染模組統一使用 G_userAnswers
   - 提交/計分模組從 G_userAnswers 讀取

5. 智慧解析引擎：從 API 載入法規資料
   - 支援 25 部法規查詢
   - 支援 pcode 和名稱查詢
   - 快取機制已啟用
   - 解析摘要、答題秘訣、邏輯標籤自動生成

6. 模組化載入：依序載入 A→B→C→D 區塊
   - A 區塊（00~09）：基礎架構
   - B 區塊（10~14）：解析引擎
   - C 區塊（20~26）：系統控制
   - D 區塊（30~36）：題型渲染
