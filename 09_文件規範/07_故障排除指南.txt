【一】頁面無法載入

症狀：開啟系統主頁面.html 後一片空白

排查步驟：
  1. 檢查檔案路徑是否正確
  2. 確認所有模組檔案是否存在（03_核心邏輯/）
  3. 開啟瀏覽器開發者工具（F12）查看 Console 錯誤

解決方案：
  - 正確路徑為 ../03_核心邏輯/、../02_前端服務/、../01_前端資源/
  - 若為模組缺失，從版本備份恢復備份

【二】題目無法載入

症狀：載入 0 題或持續載入中

排查步驟：
  1. 檢查後端是否正常運作：http://localhost:8001/api/health
  2. 檢查 Console 是否有錯誤訊息

解決方案：
  - 若後端未啟動：cd 05_後端伺服器 && python 00_後端入口.py
  - 檢查 Supabase 連線是否正常

【三】圖片無法顯示（v10.1.1）

症狀：單選題、複選題、連連看題目應顯示圖片但未顯示

排查步驟：
  1. 確認圖片檔案是否存在於正確路徑
     - 單選題：01_前端資源/06_題庫圖片/01_單選題/S001.png
     - 複選題：01_前端資源/06_題庫圖片/02_複選題/M001.png
     - 連連看：01_前端資源/06_題庫圖片/03_連連看/L001.png
  2. 確認圖片檔案名稱是否與題目 ID 完全一致

解決方案：
  - 若圖片不存在，將圖片放入對應目錄
  - 若圖片名稱錯誤，重新命名為正確的題目 ID

【四】官方格式顯示異常（v10.1.1）

症狀：題型顯示不符合官方格式

排查步驟：
  1. 確認 D 區塊渲染器是否為最新版本
     - 30_渲染_單選複選.js → v3.2
     - 31_渲染_是非填空.js → v2.0
     - 32_渲染_計算題.js → v2.1
     - 33_渲染_配合題.js → v2.0
     - 34_渲染_排序題.js → v2.0
     - 35_渲染_連連看.js → v3.2

解決方案：
  - 若渲染器版本過舊，從版本備份恢復最新版本
  - 清除瀏覽器快取（Ctrl + Shift + Delete）

【五】後端無法啟動

症狀：執行 python 00_後端入口.py 後報錯

排查步驟：
  1. 確認 Python 依賴已安裝：pip install -r requirements.txt
  2. 確認 Supabase 連線資訊正確
  3. 確認 .env 檔案存在且格式正確

解決方案：
  - 安裝依賴：pip install fastapi uvicorn sqlalchemy psycopg2-binary python-dotenv pydantic
  - 檢查 DATABASE_URL 格式：postgresql://user:password@host:5432/postgres

【六】智慧化解析區無法顯示

症狀：解析區空白或顯示「無解析內容」

排查步驟：
  1. 確認題目是否有 logic、tips、explanation 欄位
  2. 檢查 11_解析_秘訣生成模組.js 是否正確載入
  3. 檢查 14_解析_主引擎.js 是否正確載入

解決方案：
  - 若 logic 為空，執行 10_工具腳本/高端精煉_自動修正腳本.py
  - 若 tips 為空，檢查 11_解析_秘訣生成模組.js
  - 確認 14_解析_主引擎.js 有正確載入法規資料庫

【七】公式未顯示

症狀：計算題未顯示公式

排查步驟：
  1. 確認題目是否有 formulaKey 欄位
  2. 檢查 20_系統_全域變數.js 中的 FORMULA_MAP 是否包含該 formulaKey

解決方案：
  - 若 FORMULA_MAP 缺少該公式，執行 10_工具腳本/擴充_FORMULA_MAP.py

【八】法規資料庫載入失敗（v10.1.1 新增）

症狀：解析時無法顯示法規條文內容

排查步驟：
  1. 確認 06_資料庫/07_法規資料庫/ChLaw_完整版.json 是否存在
  2. 確認 14_解析_主引擎.js 中的載入路徑是否正確
  3. 檢查 JSON 格式是否正確

解決方案：
  - 確認路徑：../06_資料庫/07_法規資料庫/ChLaw_完整版.json
  - 若檔案損壞，從備份恢復
  - 若格式錯誤，使用 JSON 驗證器檢查

【九】知識庫載入失敗（v10.1.1 新增）

症狀：知識庫功能無法使用

排查步驟：
  1. 確認知識庫檔案是否存在
  2. 檢查瀏覽器 Console 錯誤訊息

解決方案：
  - 確認 06_資料庫/06_完整題庫/ 中所有 JSON 檔案完整
  - 確認 06_資料庫/07_法規資料庫/ChLaw_完整版.json 存在
  - 重新整理頁面（Ctrl + F5）

【十】G_EXAM_DATE is not defined

症狀：Console 出現此錯誤

原因：20_系統_全域變數.js 未正確載入

解決方案：
  1. 確認 系統主頁面.html 中 20 在 21 之前載入
  2. 確認 20_系統_全域變數.js 存在於 03_核心邏輯/

【十一】renderQuestion is not defined

症狀：Console 出現此錯誤

原因：36_渲染_主入口.js 未正確載入

解決方案：
  1. 確認 系統主頁面.html 中已載入 36_渲染_主入口.js
  2. 確認 36_渲染_主入口.js 存在於 03_核心邏輯/

【十二】快速檢查清單

遇到問題時，依序檢查：
  □ 瀏覽器 Console 是否有紅色錯誤
  □ 所有模組是否正確載入
  □ 檔案路徑是否正確（../03_核心邏輯/、../02_前端服務/）
  □ 後端 API 是否正常運作（http://localhost:8001/api/health）
  □ 圖片檔案是否存在於正確路徑（v10.1.1）
  □ 法規資料庫是否完整（ChLaw_完整版.json）
  □ 知識庫檔案是否完整
  □ 瀏覽器快取是否已清除
